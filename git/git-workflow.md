# Git Workflow

## Standard change flow

1. Update the local main branch.
2. Create a dedicated feature branch.
3. Make and validate the change.
4. Commit the change locally.
5. Push the branch to the remote repository.
6. Open a Merge Request.
7. Pass CI checks and code review.
8. Merge the change into main.
9. Deploy and verify system health.

## Operational principle

A successful merge does not guarantee a successful deployment.

A successful deployment does not guarantee application health.

Application health does not guarantee that the business flow works correctly.
